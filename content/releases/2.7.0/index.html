Release Notes

This release contains a small set of new features and a number of important bug fixes.

New Features

  • [DAFFODIL-1114] DPath: fn:namespace-uri() Unsupported Function
  • [DAFFODIL-341] variables - newVariableInstance
  • [DAFFODIL-357] Asserts with failureType=”recoverableError” to create warnings

Bug Fixes

  • [DAFFODIL-1101] MissingFormatArgumentException: Format specifier ‘s’
  • [DAFFODIL-1656] Writing to a variable multiple times does not cause an SDE during parsing
  • [DAFFODIL-1681] Need validation of infoset data
  • [DAFFODIL-2120] dfdl:textStringPadCharacter not validated/cooked properly. It is set to “ “ in AF.dfdl.xsd but should be %SP;
  • [DAFFODIL-2124] Error unparsing bigInts
  • [DAFFODIL-2199] initiatedContent=”yes” does not cause SDE when initiator=”%ES;”
  • [DAFFODIL-2327] expresion on all unsignedInt element and variables results in cast exception
  • [DAFFODIL-2328] setValidation - bug. Is calling super.withValidation which will not work.
  • [DAFFODIL-2334] Mutable Variable API changes break schemas
  • [DAFFODIL-2338] Use consistent validation of primitives
  • [DAFFODIL-2342] Variable issue with LINK16 schema project
  • [DAFFODIL-2349] totalDigits valiation broken when totalDigits >= 10

Improvements

These changes improve code maintainability.

  • [DAFFODIL-2187] too many debug/oolagdebug messages from sbt it:test
  • [DAFFODIL-1326] RuntimeData objects should not carry namespaces
  • [DAFFODIL-1447] Performance: Improve performance with VariableMap
  • [DAFFODIL-1673] code quality: use non-deprecated syntax for methods that don’t return a value
  • [DAFFODIL-1983] Eliminate eclipse classpath maintenance - UpdateEclipseClasspaths - run from sbt
  • [DAFFODIL-2303] catch of Throwable in ExternalVariablesLoader.scala
  • [DAFFODIL-2310] Move VariableMap.scala into runtime1 package
  • [DAFFODIL-2325] Eliminate complexity of by-Name parameter passing to RuntimeData objects
  • [DAFFODIL-2355] Update to Apache license for Scala dependencies

TDML

  • [DAFFODIL-2340] TDML Runner suppresses loading warnings from invalid config files

Miscellaneous Changes and Bug Fixes

  • [DAFFODIL-2355] Update to Apache license for Scala dependencies

Incubation is required of all newly accepted projects until a further review indicates that the infrastructure, communications, and decision making process have stabilized in a manner consistent with other successful ASF projects. While incubation status is not necessarily a reflection of the completeness or stability of the code, it does indicate that the project has yet to be fully endorsed by the ASF.
